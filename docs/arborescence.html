UpcycleConnect est une plateforme d'économie circulaire permettant à des utilisateurs de publier des annonces de réemploi, des prestations de service, et de participer à des événements. L'architecture est en monorepo avec un backend API REST en Go et un frontend SPA en Vue 3. L'ensemble est orchestré via Docker Compose avec un reverse proxy Apache.

Infrastructure — apache/
Reverse proxy Apache entre les clients et les services internes
Architecture réseau : Client → Apache:80 → /api/* → Backend Go:8080 | /* → Frontend Vue:80
FichierRôle
Dockerfile Image Apache basée sur httpd:latest. Copie vhost.conf dans la configuration Apache et active les modules proxy.
vhost.conf VirtualHost Apache. Routes /api/* et /uploads/* vers le backend Go. /api/public/v1/* exposé sans authentification. Tout le reste est servi par le frontend Vue.

Backend Go — backend/
API REST en Go avec Gin, GORM, JWT — Port interne 8080
Point d'entrée et configuration
FichierRôle
main.go Point d'entrée de l'application. Charge la config, ouvre la connexion DB (GORM auto-migrate), configure le router Gin, démarre le serveur HTTP sur :8080.
go.mod Module Go upcycleconnect/backend. Déclare les dépendances : gin-gonic/gin, gorm.io/gorm, golang-jwt/jwt, go-sql-driver/mysql, joho/godotenv.
go.sum Checksums cryptographiques de toutes les dépendances. Garantit l'intégrité des modules téléchargés.
Dockerfile Build multi-stage : étape builder compile le binaire Go, étape finale copie uniquement le binaire dans une image Alpine légère.
config/config.go Charge et expose la configuration depuis les variables d'environnement : DB credentials, JWT secret, port, clé API SIRET (Société.com), SMTP.
database/database.go Initialise la connexion GORM/MySQL. Configure le pool de connexions. Lance AutoMigrate sur tous les models pour synchroniser le schéma.
router/router.go Enregistre toutes les routes API. Trois groupes : /api/public/v1 (public), /api/v1 (utilisateurs), /api/admin/v1 (admins). Applique les middlewares CORS, Auth, RBAC.

github.com/gin-gonic/gin v1.10.0 Framework HTTP principal. Gère le routing (groupes, paramètres, wildcards), le binding JSON/form des requêtes, les middlewares chaînés, la récupération sur panic, et la sérialisation des réponses JSON. Toutes les routes de l'API sont définies avec Gin.
github.com/golang-jwt/jwt/v5 v5.2.1 Génération et validation des tokens JWT (JSON Web Tokens). Utilisé dans handlers/auth.go et handlers/user_auth.go pour créer les tokens de session signés HS256, et dans middleware/auth.go pour les valider à chaque requête protégée.
github.com/gosimple/slug v1.14.0 Génération de slugs URL à partir de chaînes Unicode. Utilisé dans handlers/categories.go pour créer automatiquement le slug d'une catégorie à partir de son nom (ex : "Mobilier Recyclé" → mobilier-recycle).
golang.org/x/crypto v0.23.0 Algorithmes cryptographiques étendus. Utilisé exclusivement pour bcrypt : hachage des mots de passe lors de l'inscription (bcrypt.GenerateFromPassword) et vérification lors du login (bcrypt.CompareHashAndPassword).
gorm.io/driver/mysql v1.5.7 Driver MySQL pour GORM. Construit la chaîne DSN (Data Source Name) et établit la connexion entre GORM et le serveur MySQL. Utilisé dans database/database.go via mysql.Open(dsn).
gorm.io/gorm v1.25.10 ORM (Object-Relational Mapper) principal. Gère toutes les interactions avec la base de données : AutoMigrate pour synchroniser les tables depuis les structs Go, requêtes CRUD (Find, Create, Save, Delete), relations (has-many, belongs-to), pagination, filtres dynamiques et soft-delete.
